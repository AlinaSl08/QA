# Чек‑лист

**Чек‑лист (Checklist)** – это структурированный перечень пунктов (областей, функций, сценариев), которые необходимо проверить в процессе тестирования ПО.

**Структура чек‑листа**: 

|Поле|	Описание|	Пример|
|:--:|:--:|:--:|
|ID|	Уникальный идентификатор пункта чек‑листа	111111|
|Раздел/Модуль|	К какой части системы относится проверка|	Авторизация, Корзина покупок|
|Пункт проверки|	Что именно нужно проверить|	Вход с валидными логином и паролем, Удаление товара из корзины|
|Статус	|Результат проверки (Passed/Failed/Blocked)	|Passed|
 

**Пошаговая инструкция составления чек-листа**: 

**Шаг 1. Определите область тестирования**

+ Выберите модуль/функцию (например, "Корзина покупок", "Авторизация").

**Шаг 2. Выдели основные сценарии**

+ Перечислите ключевые действия пользователя (добавить товар в корзину, изменить количество товаров в корзине, удалить товар из корзины).

**Шаг 3. Сгруппируйте пункты**

+ Объедините связанные проверки (например, все операции с корзиной).

**Шаг 6. Проверьте полноту**

+ Убедитесь, что охвачены все ключевые сценарии.
+ Исключите дублирование.

|:--:|:--:|:--:|:--:|

|234245|	Авторизация	|Ошибка при пустом поле email	|Passed|
|234246|	Авторизация|	Ошибка при неверном пароле	|Passed|
|234247|	Авторизация|	Восстановление пароля (отправка письма)	|Failed|
|234248|	Авторизация|	Ограничение попыток входа (3 неудачных = блокировка)	|Passed|
|234249|	Корзина|	Добавление товара в корзину	|Passed|
|234250	|Корзина|	Изменение количества товара в корзине	|Failed|
