# API

**API** (Application Programming Interface) – это набор правил, по которым программы взаимодействуют друг с другом.

**Суть**: API определяет:

+ какие запросы можно отправлять;
+ в каком формате должны быть данные;
+ какие ответы ожидаются;
+ как обрабатывать ошибки.

**Примеры из жизни**:

+ мобильное приложение запрашивает погоду у метеосервиса;
+ сайт показывает карту через API картографического сервиса;
+ платёжная система передаёт данные в банк.

**2. Основные виды API**
**1. REST (Representational State Transfer)**

+ основан на HTTP;
+ использует стандартные методы (`GET`, `POST`, `PUT`, `PATCH`, `DELETE`);
+ ресурсы адресуются через URL;
+ самый популярный стиль для веб‑сервисов.

**2. SOAP (Simple Object Access Protocol)**

+ строгая спецификация;
+ данные в XML;
+ поддерживает сложные транзакции;
+ часто используется в корпоративных системах.

**3. GraphQL**

+ разработан Facebook;
+ клиент сам указывает, какие данные нужны;
+ один запрос может вернуть данные из разных источников;
+ эффективен для сложных интерфейсов.

**4. RPC (Remote Procedure Call)**

+ вызов функций на удалённом сервере;
+ примеры: JSON‑RPC, XML‑RPC;
+ прост для небольших задач.

**5. WebSocket API**

+ двусторонняя связь в реальном времени;
+ подходит для чатов, онлайн‑игр, мониторинга.
