# Тест-кейс

**Тест-кейс (Test Case)** – это подробное описание последовательности действий, условий и ожидаемого результата, разработанное для проверки определенной функциональности программного обеспечения.

 

**Виды тест‑кейсов**:

**1. Позитивные**

+ Проверяют корректную работу при валидных данных.
+ Пример: вход в систему с правильным логином и паролем.

**2. Негативные**

+ Тестируют обработку ошибок (невалидные данные).
+ Пример: попытка входа с пустым паролем.
 

**Поля тест-кейса:**

|Поле|	Описание|	Пример|
|:--:|:--:|:--:|
|**ID**|	Уникальный идентификатор тест-кейса|	TC-001 (может быть любой)|
|**Заголовок (Title)**|	Краткое описание того, что проверяется|	Проверить авторизацию с валидным email и паролем|
|**Предусловия (Preconditions)**|	Что должно быть выполнено до начала теста|	Пользователь зарегистрирован. Приложение открыто.|
|**Окружение (Environment)**|	Информация об устройстве, ОС, браузере, версии приложения и т.д.|	ОС: Windows 11, Браузер: Chrome 128, Версия: 2.1.0|
|**Шаги выполнения (Test Steps)** |Последовательность действий в тесте (обычно пронумерована)|1. Открыть страницу входа 2. Ввести email 3. Ввести пароль 4. Нажать "Войти"|
|**Ожидаемый результат (Expected Result)**|	Что должно произойти после выполнения шагов|	После нажатия "Войти" пользователь попадает в личный кабинет|
|**Статус (Status)**|	Заполняется после выполнения теста|	Passed / Failed / Blocked|
 

**Основные статусы выполнения тест-кейса**:
1. **Passed (Пройден / Успешно)**: Выполнение шага или тест-кейса завершилось успешно;
2. **Failed (Провален / Неуспешно)**: Выполнение шага или тест-кейса завершилось с ошибкой (баг);
3. **Blocked (Заблокирован)**: Этот статус означает, что тест-кейс не может быть выполнен, потому что какое-то внешнее условие мешает его   выполнению. Самая частая причина — это наличие другого, критического бага, который не дает добраться до нужного функционала.

 

**Пример тест-кейса:**

**ID**: TC-CART-001

**Заголовок**: Проверить добавление и изменение количества товара в корзине

**Окружение**: ОС: Windows 11, Браузер: Яндекс Браузер, Версия: 1.1.0

**Предусловия**: Пользователь авторизован, в наличии товар "Смартфон A".

|№ шага| Действие| Ожидаемый результат |Статус шага|
|:--:|:--:|:--:|:--:|
|1|	Перейти в каталог товаров|	Отображается список товаров с ценами и кнопками "В корзину".|	Passed|
|2|	Нажать "В корзину" у "Смартфон A"|	1 товар добавлен в корзину. Появляется уведомление об успешном добавлении.|	Passed|
|3|	Открыть корзину|	Отображается страница корзины, в списке есть "Смартфон A", количество: 1, итоговая цена корректна.	|Passed|
|4|	Изменить количество на "3", нажать "Обновить"|	Количество товара изменяется на 3. Итоговая цена пересчитывается корректно.|	Failed|
|5|	Проверить иконку корзины в шапке сайта|	На иконке отображается число "3" (общее количество товаров).	|Failed|
|6|	Нажать "Удалить"|	Товар удаляется из корзины. Отображается сообщение "Корзина пуста".|	Passed|

**Итоговый статус тест-кейса**: Failed (так как шаги 4 и 5 не были пройдены)
