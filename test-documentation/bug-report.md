# Тестовая документация

**Тестовая документация** – это набор записей и инструкций, создаваемых в процессе тестирования программного обеспечения. К основной документации относятся: 

1. Баг-репорт
2. Тест-кейс
3. Чек-лист
4. Отчёт по тестированию
 
# Баг-репорт

**Баг-репорт** – это документ, описывающий найденную ошибку в программе. Он фиксирует, что работает не так, как ожидалось. Включает в себя следующие поля: 

|Поле| Описание| Пример значения|
|:--:|:--:|:--:|
|**ID**	|Уникальный номер в системе (генерируется автоматически)|	BUG-12345|
|**Заголовок (Title/Summary)**|	Краткое описание сути проблемы	|Кнопка "Войти" неактивна при вводе валидных данных|
|**Предусловия**|	Что нужно подготовить для воспроизведения бага|	Зарегистрировать пользователя|
|**Шаги воспроизведения (Steps to Reproduce)**|	Точная последовательность действий, чтобы увидеть баг|	1. Открыть страницу входа... 2. Ввести данные...|
|**Фактический результат (Actual Result)**|	Что происходит на самом деле|	Кнопка "Войти" серая и не нажимается.|
|**Ожидаемый результат (Expected Result)**|	Как должно работать по логике/требованиям|	Кнопка "Войти" должна быть активна.|
|**Серьёзность (Severity)**|	Степень влияния на работу системы (Trivial, Minor, Major, Critical, Blocker)|	Critical|
|**Приоритет (Priority)**|	Срочность исправления (High, Medium, Low)|	High|
|**Вложения (Attachments)**|	Скриншоты, видео, логи|	screenshot_1.png|
 

**Пример баг-репортера:**

**ID**: BUG-001

**Заголовок**: При попытке оплаты картой через СБП возникает ошибка 500 (Internal Server Error) и платеж не проходит

**Серьёзность (Severity)**: Blocker (Блокирует основной функционал)

**Приоритет (Priority)**: High (Требует немедленного исправления)

**Предусловия**:

+ В корзине интернет-магазина находится хотя бы один товар.
+ Пользователь авторизован.

**Шаги воспроизведения:**
1. Перейти в корзину товаров.
2. Нажать кнопку "Оформить заказ".
3. На странице выбора способа оплаты выбрать "Система быстрых платежей (СБП)".
4. Нажать кнопку "Оплатить".
5. Дождаться перенаправления на страницу оплаты.


**Фактический результат**:
На странице оплаты отображается ошибка: Оплата невозможна.

**Ожидаемый результат**:
Пользователь должен быть успешно перенаправлен на страницу оплаты СБП, где отображается QR-код для сканирования и завершения платежа.
